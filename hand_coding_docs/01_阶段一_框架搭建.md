## 1. app/config.py - 配置管理

### 文件作用
集中管理所有配置项，包括：
- 应用基础配置（名称、版本、端口）
- LLM 配置（API Key、模型名称）
- Milvus 配置（主机、端口）
- RAG 配置（检索参数）
- MCP 服务配置（外部工具服务）

### 为什么要先写
- 所有其他模块都依赖配置，它是基础设施中的基础设施
- 使用 Pydantic Settings 可以实现类型安全的配置管理
- 从 `.env` 文件读取配置，便于不同环境切换

### 手敲代码（先写这些）

```python
# app/config.py
from typing import Dict, Any
from pydantic_settings import BaseSettings, SettingsConfigDict


class Settings(BaseSettings):
    """应用配置"""

    model_config = SettingsConfigDict(
        env_file=".env",
        env_file_encoding="utf-8",
        case_sensitive=False,
        extra="ignore",
    )

    # 应用配置
    app_name: str = "SuperBizAgent"
    app_version: str = "1.0.0"
    debug: bool = False
    host: str = "0.0.0.0"
    port: int = 9900

    # DashScope 配置
    dashscope_api_key: str = ""
    dashscope_model: str = "qwen-max"
    dashscope_embedding_model: str = "text-embedding-v4"

    # Milvus 配置
    milvus_host: str = "localhost"
    milvus_port: int = 19530
    milvus_timeout: int = 10000

    # RAG 配置
    rag_top_k: int = 3
    rag_model: str = "qwen-max"

    # 文档分块配置
    chunk_max_size: int = 800
    chunk_overlap: int = 100

    # MCP 服务配置
    mcp_cls_transport: str = "streamable-http"
    mcp_cls_url: str = "http://localhost:8003/mcp"
    mcp_monitor_transport: str = "streamable-http"
    mcp_monitor_url: str = "http://localhost:8004/mcp"

    @property
    def mcp_servers(self) -> Dict[str, Dict[str, Any]]:
        """获取完整的 MCP 服务器配置"""
        return {
            "cls": {
                "transport": self.mcp_cls_transport,
                "url": self.mcp_cls_url,
            },
            "monitor": {
                "transport": self.mcp_monitor_transport,
                "url": self.mcp_monitor_url,
            }
        }


# 全局配置实例
config = Settings()
```

### 调用关系
- **被谁调用**：所有需要配置的模块（main.py, services, tools 等）
- **调用谁**：无

### 初学者最容易误解的地方
1. **环境变量优先级**：`.env` 文件的值优先于代码中的默认值
2. **case_sensitive=False**：环境变量不区分大小写，但建议使用大写
3. **@property** 方法的使用：`mcp_servers` 不是从环境变量读取，而是根据其他配置动态生成的

### 写完后应掌握
- Pydantic Settings 的使用方法
- 如何从 `.env` 文件读取配置
- 全局单例模式的实现（`config = Settings()`）

## 2. app/utils/logger.py - 日志配置

### 文件作用
配置 Loguru 日志库，实现：
- 日志文件按天轮转
- 控制台和文件同时输出
- 统一的日志格式

### 为什么要先写
- 开发和调试都需要日志输出
- 日志应该在应用启动时就配置好
- `app/__init__.py` 会自动导入它

### 手敲代码

```python
# app/utils/logger.py
from loguru import logger
import sys

# 移除默认的处理器
logger.remove()

# 添加控制台处理器（带颜色）
logger.add(
    sys.stdout,
    format="<green>{time:YYYY-MM-DD HH:mm:ss}</green> | <level>{level: <8}</level> | <cyan>{name}</cyan>:<cyan>{function}</cyan>:<cyan>{line}</cyan> - <level>{message}</level>",
    level="INFO"
)

# 添加文件处理器（按天轮转）
logger.add(
    "logs/app_{time:YYYY-MM-DD}.log",
    format="{time:YYYY-MM-DD HH:mm:ss} | {level: <8} | {name}:{function}:{line} - {message}",
    level="DEBUG",
    rotation="00:00",  # 每天午夜轮转
    retention="30 days",  # 保留30天
    compression="zip",  # 压缩旧日志
    encoding="utf-8"
)
```

### 调用关系
- **被谁调用**：所有模块（通过 `from loguru import logger`）
- **调用谁**：无

### 初学者最容易误解的地方
1. **logger.remove()**：必须先移除默认处理器，否则会有重复日志
2. **rotation="00:00"**：表示每天午夜 00:00 创建新日志文件
3. **为什么在 `__init__.py` 中导入**：确保应用启动时就配置好日志

### 写完后应掌握
- Loguru 的基本用法
- 如何配置多个日志处理器
- 日志轮转和保留策略

## 3. app/main.py - FastAPI 入口

### 文件作用
FastAPI 应用的入口文件，负责：
- 创建 FastAPI 应用实例
- 配置 CORS 中间件
- 注册路由
- 挂载静态文件
- 应用生命周期管理（启动/关闭）

### 为什么要先写
- 它是整个应用的入口，其他所有功能都挂载在上面
- 需要在此配置好基础设置（CORS、中间件等）

### 手敲代码（分部分写）

**第一部分：导入和生命周期管理**

```python
# app/main.py
from fastapi import FastAPI
from fastapi.middleware.cors import CORSMiddleware
from contextlib import asynccontextmanager

from app.config import config
from loguru import logger


@asynccontextmanager
async def lifespan(app: FastAPI):
    """应用生命周期管理"""
    # 启动时执行
    logger.info("=" * 60)
    logger.info(f"🚀 {config.app_name} v{config.app_version} 启动中...")
    logger.info(f"📝 环境: {'开发' if config.debug else '生产'}")
    logger.info(f"🌐 监听地址: http://{config.host}:{config.port}")
    logger.info(f"📚 API 文档: http://{config.host}:{config.port}/docs")

    # TODO: 后续添加 Milvus 连接等初始化代码

    logger.info("=" * 60)

    yield

    # 关闭时执行
    logger.info(f"👋 {config.app_name} 关闭")
```

**第二部分：创建应用和配置中间件**

```python
# 创建 FastAPI 应用
app = FastAPI(
    title=config.app_name,
    version=config.app_version,
    description="基于 LangChain 的智能运维系统",
    lifespan=lifespan
)

# 配置 CORS
app.add_middleware(
    CORSMiddleware,
    allow_origins=["*"],
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)
```

### 调用关系

- **调用谁**：config, 各 API 路由模块

### 初学者最容易误解的地方
1. **lifespan 上下文管理器**：`yield` 之前是启动逻辑，之后是关闭逻辑
2. **CORS 中间件**：生产环境应该限制 `allow_origins` 为具体域名，而非 `"*"`
3. **路由注册顺序**：路由的注册顺序不影响匹配，但建议按功能分组
